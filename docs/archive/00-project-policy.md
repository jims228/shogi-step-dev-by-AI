# A. プロジェクトポリシー: Reference-Driven Clean-Room Rebuild

## 旧ポリシー（修正前）
> 旧リポジトリは参照のみ。実装ベースにしない。

## 新ポリシー
> **Reference-Driven Clean-Room Rebuild（参照駆動型クリーンルーム再構築）**
>
> 旧リポジトリを**積極的に調査**し、プロダクト設計・アーキテクチャ・UX・データモデリングの知見を**抽出**した上で、実装は**ゼロから新規に書く**。

### ルール
1. 各セッションは実装前に**旧リポジトリの該当部分を必ず調査**する
2. 調査結果から**原則・パターン・教訓**を抽出する
3. 実装は旧コードをコピペせず**新規に書く**
4. 旧構造を「すでにあるから」という理由だけで引き継がない
5. 旧構造が**良い理由が説明できる**場合のみ、同じアプローチを採用してよい
6. 旧構造に**問題があった部分**は明示的に避ける
